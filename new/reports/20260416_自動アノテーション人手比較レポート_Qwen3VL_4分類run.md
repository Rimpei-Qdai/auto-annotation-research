## 7. 原因整理

### 原因 1. 4分類 prompt に対してモデルの重心が `C` と `D` に寄っている

今回の出力は `A/B/C/D` の 4択だが、実際には

- `B` が 4 件しか出ていない
- `C` が 30 件
- `D` が 41 件

となっており、Qwen3-VL が「直線」よりも「右回転」または「その他」を選びやすい傾向を示している。

特に `A -> C` が 20 件、`A -> D` が 19 件あるため、直進寄りの scene を安定して `A` に置けていない。

### 原因 2. 軌道描画付き 4 フレーム入力では、軽微な横ずれを回転として誤読している可能性が高い

今回の比較は Qwen2 系と同じ prompt / 同じ入力様式に寄せているため、各フレームに軌道を重畳したまま 4 枚を見せている。

この形式では

- 道路背景
- 車両周辺の scene
- 軌道の赤線
- 緑の基準線

が同一画像内で競合し、わずかな横ずれや road curvature を「右回転」と読んでいる可能性がある。

### 原因 3. `D` は stop-like / low-speed 系をまとめて受け止めるため、逃げ先になりやすい

今回の macro 4分類では

- 停止
- 発進
- その他

がすべて `D` に入るため、モデルにとって曖昧な sample の受け皿になりやすい。

その結果、`A -> D` が 19 件と大きくなっている。

### 原因 4. このブランチでは推論過程ログが十分に残っていない

[annotation.log](/Users/rimpeihata/Desktop/auto-annotation-research/new/annotation_tool/kiwi_logs/annotation.log) には今回 run の frame extraction までは見えるが、`Model output:` や sample ごとの raw generated text が残っていない。

また [inference_process.jsonl](/Users/rimpeihata/Desktop/auto-annotation-research/new/annotation_tool/kiwi_logs/inference_process.jsonl) は今回 run の内容ではなく古い run の内容のままであり、今回は raw prompt / raw output ベースの誤り分析ができていない。

これは解析上の大きな制約である。

## 8. 改善案

### 改善案 A. Qwen3-VL 比較を続けるなら、まずログを復活させる

優先度: 最優先

- `prompt_text`
- `generated_text`
- `sample_id`
- `predicted_macro`
- `trajectory_visible_points`

を今回のブランチにも保存する。

今のままだと「なぜ `C` を選んだか」を sample 単位で追えない。

### 改善案 B. 4分類の比較では、元フレームと軌道 summary を分離した入力へ戻す

優先度: 高

今回の run は「Qwen2 時代と同じ prompt / 同じ入力様式」に寄せた比較だったが、その形式自体が Qwen3-VL にとって軌道を読みにくくしている可能性が高い。

次の比較候補:

- 画像 1〜4: 元フレーム
- 画像 5: 軌道 summary

とし、軌道を別画像として明示する。

### 改善案 C. まず `A vs 非A` の二段階 macro 判定に分ける

優先度: 高

いきなり `A/B/C/D` の 4択にすると、

- `A -> C`
- `A -> D`

が大きくなりやすい。

そのため次は

1. `A (直線系)` か `非A` か
2. `非A` なら `B/C/D`

の段階判定にすると、少なくとも直線系の吸い込み精度を上げやすい。

### 改善案 D. 回転方向だけを別 prompt で比較する

優先度: 中

今回 `B` と `C` の非対称が大きいので、回転系 sample だけを抜き出して

- 左か右か

だけを答えさせる短い prompt を別に評価すると、Qwen3-VL が本当に左右を読めていないのか、4分類 prompt 全体で崩れているのかを切り分けられる。

### 改善案 E. 4分類評価を主指標にし、11分類は補助指標に下げる

優先度: 中

今回の run は設計上 4分類モデルであるため、11分類 accuracy は構造的に低く出る。

今後この系を比較する際は

- 主指標: 4分類 Accuracy / Macro F1 / kappa
- 補助指標: 11分類 representative label accuracy

と分けて扱う方が自然である。

## 9. 結論

今回の Qwen3-VL 4分類 run は、

- 動画探索の修正により未出力は 4 件まで減少
- しかし分類精度は
  - 11分類: `35.42%`
  - 4分類: `39.58%`

に留まり、特に

- `A (直線系) -> C (右回転系)`
- `A (直線系) -> D (その他)`

の誤りが大きかった。

したがって、現時点では

- Qwen3-VL の比較は成立した
- ただし入力設計とログ設計がまだ不十分で、モデル性能を正しく使い切れていない

という整理になる。
