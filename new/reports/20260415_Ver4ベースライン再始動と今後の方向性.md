# Ver.4 ベースライン再始動と今後の方向性

**作成日**: 2026年4月15日  
**対象プロジェクト**: タクシー動画の運転行動自動アノテーション  
**目的**: `main` から切った新ブランチで Ver.4 baseline を再始動し、今後の改善方針を共有する

---

## 1. このブランチの役割

このブランチは、**Ver.4 baseline の再現と再評価** を目的とする。

Ver.4 baseline は、次の構成を指す。

- `Qwen2-VL-2B-Instruct`
- 4 フレーム入力
- 赤軌道の Visual Prompting
- 直接分類
- L2M/CoT なし

この構成を現在の評価セットで再実行し、今後の比較基準を固定する。

---

## 2. なぜ戻すのか

これまでの検証から、L2M/CoT や Graph を全件に適用すると、

- 強い視覚情報が粗い中間要約に潰れる
- センサとの矛盾が残ったまま後段へ渡る
- 最終段が文章的に再解釈して不安定になる

ことが分かった。

一方で、Ver.4 は

- 4 フレーム
- 赤軌道
- 直接分類

というシンプルな構成で、既存レポート上もっとも強い baseline だった。

したがって、今後はまず Ver.4 を再現し、その上で差分改善だけを評価する。

---

## 3. 今後の方向性

今後の改善は、**Visual 強化** と **Graph 強化** の二軸で進める。

### 3.1 Visual 強化

- VLM が見れば分かる情報を画像上で明示する
- 赤軌道、速度感、車線との関係、交差点文脈を視覚的に強化する

### 3.2 Graph 強化

- 視覚・軌道・センサから得た判断材料を整理する
- 全件の主分類器ではなく、曖昧ケースの verifier / reranker として使う

---

## 4. 実装ルール

このブランチでは、以下を原則とする。

1. 主経路は Ver.4 baseline を維持する
2. L2M/CoT は既定で無効にする
3. 改善は baseline を壊さない差分として追加する
4. 比較は必ず baseline と同条件で行う

---

## 5. 直近の作業

1. Ver.4 baseline で自動アノテーションを再実行する
2. 人手アノテーションと比較して baseline を再確認する
3. その後、Visual 強化のみ、Graph 強化のみ、Visual + Graph を順に評価する

---

## 6. 一文で言うと

**このブランチは、Ver.4 を比較の土台として復元し、今後は「Visual を主軸、Graph を補助軸」として改善を積み上げるための本流ブランチである。**
