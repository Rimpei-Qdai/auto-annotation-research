## 5. ログから見えた原因

### 5.1 停止に寄りすぎる

`sample 37-42` などでは、

- `trajectory_point_like=True`
- `trajectory_visual_speed_state=STOPPED`
- `speed_state=STOPPED`
- `stop_likelihood=HIGH`

となっており、人手は `等速走行` だが自動は `停止` に倒れていた。  
特に `speed=0` でも `brake=OFF` のケースを停止へ寄せすぎていた。

### 5.2 左右折に寄りすぎる

`sample 48, 66, 72, 99, 100` や `sample 23, 53, 63, 80, 92` では、

- 赤軌道の左右キュー
- `intersection_detected=YES` または `UNCERTAIN`

が turn を強く押していた。  
ただし、今回の run では `intersection_detected` が

- `YES`: 62件
- `UNCERTAIN`: 37件
- `NO`: 0件

となっており、交差点文脈を信じすぎていた。

### 5.3 Level 3 の最終判断が turn に寄りやすい

`sample 23` や `sample 85` では、Graph 上は `加速` や `減速` が上位でも、Level 3 が `右折` を返していた。  
このため、Graph の speed-event 候補を turn より優先する拘束が必要だった。

## 6. 今回実装した改善

### 6.1 交差点判定の安定化

`new/annotation_tool/heron_l2m_pipeline.py`

- `intersection_detected` をそのまま信じず、`raw YES` でも以下なら `NO` または `UNCERTAIN` に落とすようにした
  - `STRAIGHT_CUE + PARALLEL + STRAIGHT + NO_SHIFT`
  - `LEFT/RIGHT_LANE_CHANGE_CUE`
  - stop-like な直進軌道
- 強い turn 文脈が揃ったときだけ `YES` を維持するようにした

### 6.2 turn より speed-event を優先する拘束

`new/annotation_tool/heron_l2m_pipeline.py`

- Level 3 の後段で、Graph 上位候補が `加速/減速/停止/等速/発進` で、turn より十分強い場合は上書きするようにした
- post-process でも同様の補正を追加した

### 6.3 stop-like 軌道の効かせ方を保守化

`new/annotation_tool/driving_graph.py`

- `point-like + STOPPED` だけで `STOPPED/HIGH` にしないようにした
- `brake=OFF` の低速・点状軌道は、停止ではなく `CONSTANT` 寄りに残しやすくした
- `stop_likelihood=MEDIUM` の加点を下げ、`HIGH` との差を広げた

### 6.4 ウィンカーなし・交差点確証なしの turn を減衰

`new/annotation_tool/driving_graph.py`

- `signal=OFF`
- `speed_state=CONSTANT`
- `intersection_state != YES`

のときは、左右折候補を減衰するようにした

## 7. テスト

以下を実行し、通過を確認した。

python3 -m py_compile new/annotation_tool/heron_l2m_pipeline.py new/annotation_tool/driving_graph.py new/annotation_tool/tests/test_l2m_pipeline.py

今回追加・更新した主な確認項目:

- straight / stop-like シーンでは `raw YES` の交差点判定を `NO` に落とす
- Graph が lane change を強く支持するとき、turn を上書きできる
- Graph が speed-event を強く支持するとき、turn を上書きできる
- `point-like + low speed + brake OFF` を即停止にしない
