## 解釈

今回の「完了済みアノテーション」は、最新 run をそのまま評価すると誤解します。

正しい読み方は次の通りです。

1. 最新 run はシステム障害で無効
2. 最後の正常 run では、停止・右折・左車線変更に改善がある
3. ただし減速・加速・左折は依然弱い
4. graph 導入版は、ロジック以前にデバイス不一致で落ちているため、現時点では性能比較の土俵に乗っていない

## 次のアクション

優先度順にやるべきことは次です。

1. `cpu/cuda:0` のデバイス不一致を直した状態で再実行する
2. 推論エラー時に `0: その他` を成功保存しない
3. 再実行後、Run 1 と同じ指標で再評価する
4. 左右旋回と停止バイアスを重点的に誤分類分析する

特に 2 は重要です。今のままだと「推論失敗」が「その他予測」に見えてしまい、精度評価が壊れます。
