- `B/C/D` がすべて出るようになった

- Stage2 の左右判定も `L=16, R=13` と左右に分かれた

## 3. まだ良くない点

### 3.1 11分類では macro 内の潰れが大きい

今回の出力は `1, 4, 6, 7` の4ラベルだけで、
`2, 3, 5, 8, 9, 10` はまだ出ていない。

そのため、たとえば以下の誤りが残っている。
- `3 -> 1`: 6件
- `1 -> 4`: 7件
- `1 -> 6`: 4件
- `1 -> 7`: 4件

つまり、macro 分類はかなり改善したが、fine label はまだ粗い。

### 3.2 直線系を停止へ寄せすぎる例が残る

`A -> D` の誤りが 10件あり、
人手で直線系のものを stop/other 側へ倒しているサンプルがある。

低速や visible points の少なさに敏感すぎる可能性がある。

### 3.3 左右回転系の中で lane change と turn を区別できていない

今回 `B/C` は出ているが、11分類に戻すと左回転はすべて `6`、右回転はすべて `7` に写像されている。
そのため `8/9`（車線変更）や `10`（転回）はまだ拾えていない。

## 4. 原因整理

### 4.1 VLM を外したことで macro 判定は安定した

今回の一番大きい知見は、
**このタスクでは VLM を主判定器から外した方が安定した** という点である。

VLM を使っていたときは、
- `1` 一極化
- `A` 一極化
- `L` 一極化

が何度も起きていた。
一方、trajectory 幾何 + speed / acc_x / gyro_z を使った deterministic 分類では、少なくとも 4分類は安定して分かれるようになった。

### 4.2 まだ macro 内の fine label 規則が不足している

現状の deterministic 実装は、4分類を安定化することを優先したため、
- 直線系の中の `等速 / 加速 / 減速`
- その他の中の `停止 / 発進 / その他`
- 回転系の中の `turn / lane change / U-turn`

を十分に分けていない。

そのため 11分類ではまだ headroom が大きい。

## 5. 改善案

### 優先度A
- macro 判定は現行 deterministic を維持する
- macro ごとに fine label の deterministic subclassifier を追加する

### 優先度B
- `A -> D` を減らすため、stop 判定に dwell 的な条件を追加する
- visible points だけで stop に寄せすぎない

### 優先度C
- `B/C` の中で `lane change` を分けるため、`heading_delta` と `lateral_ratio` の関係を使う
- `U-turn` は大きい heading 変化と短い前進距離で検出する

## 6. 今回このレポート後に入れた次の改善

このレポート作成と同時に、次回 run 向けに以下を実装した。

- Stage1 の turn 判定をさらに保守化
  - `heading + curvature` だけでは回転系へ送らず、`lateral_ratio` または `gyro` の anchor を要求
- macro の代表ラベル固定をやめ、macro 内で fine label を deterministic に分ける
  - `A -> 1/2/3`
  - `D -> 0/4/5`
  - `B -> 6/8`
  - `C -> 7/9/10`

そのため、次の run では 11分類の改善余地がある。
