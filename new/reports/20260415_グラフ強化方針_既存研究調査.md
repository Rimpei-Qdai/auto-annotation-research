# グラフ強化方針（既存研究調査）

**作成日**: 2026年4月15日  
**対象プロジェクト**: タクシー動画の運転行動自動アノテーション  
**目的**: 既存研究を踏まえ、現在の軽量 Graph をどのように強化すべきかを整理する

---

## 1. 結論

今の Graph は、

- 概念抽出
- support / contradiction の付与
- ラベル候補のスコアリング

としては有効だが、**本格的な graph reasoning には達していない**。  
既存研究を踏まえると、次に強化すべき方向は以下の 4 つである。

1. **多層 graph 化**
   - observation → concept → hypothesis → label
2. **不確実性付き concept 化**
   - 単一値ではなく confidence / uncertainty を持つ
3. **edge 主体の推論**
   - support / contradiction を説明用だけでなく、実際に候補スコアへ伝播させる
4. **交通トポロジーの graph 化**
   - lane / intersection / turn restriction を独立した subgraph として持つ

要するに、今の Graph を強くするには  
**「概念を並べて点数を付ける仕組み」から、「関係の変化が候補へ伝わる仕組み」へ進める必要がある。**

---

## 2. 現状の Graph の限界

現在の Graph は、

- `speed_state`
- `trajectory_direction`
- `intersection_state`
- `lane_crossing_state`
- `signal_state`

などの概念ノードを作り、ルールベースでラベルに加点減点している。

しかし、以下の限界がある。

### 2.1 concept が単一値に潰れている

例えば、

- `intersection_state = YES`
- `speed_state = DECELERATING`
- `trajectory_direction = LEFT`

のように、途中で概念を 1 値に確定してしまう。  
そのため、

- 「左折らしいが lane change の可能性もある」
- 「停止寄りだが減速の可能性も高い」

といった曖昧さを持ちにくい。

### 2.2 edge が説明用に近い

現在の `supports` / `contradicts` は、主に

- summary_for_prompt
- candidate_summary

に使われており、**graph propagation の中心にはなっていない**。

### 2.3 hypothesis 層がない

現在は concept から直接 label に点数を入れることが多い。  
そのため、

- `left_turn_intent`
- `left_lane_change_intent`
- `stop_event`
- `deceleration_event`

のような**中間仮説**が存在しない。

### 2.4 topology が弱い

交差点や lane change の分離に本質的な

- lane topology
- intersection topology
- turn restriction

が graph として十分に表現されていない。

## 3. 参考になる既存研究

## 3.1 DriveLM

- 論文: [DriveLM: Driving with Graph Visual Question Answering](https://arxiv.org/abs/2312.14150)

### 参考点

DriveLM は、単なる single-round VQA ではなく、

- perception
- prediction
- planning

を graph-style QA で接続することで、人間の多段推論を模倣している。  
この研究から借りられるのは、**graph を「途中の考え方を外在化する構造」として使う発想**である。

### 本研究への示唆

本研究でも、

- observation node
- concept node
- hypothesis node
- label node

の多層 graph にすることで、Graph が単なる score table から一段進む。

---

## 3.2 T2SG

- 論文: [T2SG: Traffic Topology Scene Graph for Topology Reasoning in Autonomous Driving](https://openaccess.thecvf.com/content/CVPR2025/html/Lv_T2SG_Traffic_Topology_Scene_Graph_for_Topology_Reasoning_in_Autonomous_CVPR_2025_paper.html)

### 参考点

T2SG は、lane・signal・topology の関係を scene graph として明示化し、  
交通シーンの topology reasoning を改善している。

### 本研究への示唆

本研究では、左右折と車線変更の誤判定が大きな課題である。  
そのため、Graph を強化するなら

- lane center / lane boundary
- adjacent lane availability
- intersection entry
- turn restriction / turn context

を独立ノードまたは subgraph として持つのが有効である。

つまり、今の

- `intersection_state = YES/NO`
- `lane_crossing_state = LEFT/RIGHT/NONE`

だけではなく、**交通 topology 自体を graph に入れる**必要がある。

---

## 3.3 Concept Bottleneck Models

- 論文: [Concept Bottleneck Models](https://proceedings.mlr.press/v119/koh20a.html)
- 論文: [Probabilistic Concept Bottleneck Models](https://proceedings.mlr.press/v202/kim23g.html)
- 論文: [DCBM: Data-Efficient Visual Concept Bottleneck Models](https://proceedings.mlr.press/v267/prasse25a.html)
- 論文: [Towards Safer and Understandable Driver Intention Prediction](https://openaccess.thecvf.com/content/ICCV2025/html/Karuppasamy_Towards_Safer_and_Understandable_Driver_Intention_Prediction_ICCV_2025_paper.html)

### 参考点

CBM 系は、

- 入力から直接ラベルへ行かず
- まず human-understandable concept を予測し
- その concept から最終 label を出す

発想である。  
特に ProbCBM は、concept を deterministic に 1 値で持つのではなく、**concept uncertainty** を扱う。

また、ICCV 2025 の driver intention prediction 研究は、**Video Concept Bottleneck Model (VCBM)** により、運転意図予測を concept ベースで説明可能にしている。

### 本研究への示唆

本研究で Graph を強くするなら、まず concept の持ち方を変えるべきである。

具体的には、

- `intersection_state = YES`

のような単一値ではなく、

- `intersection_prior = 0.72`
- `intersection_uncertainty = 0.18`

のように持つ方が自然である。

同様に、

- `lane_change_plausibility`
- `turn_plausibility`
- `stop_plausibility`
- `deceleration_plausibility`

を hypothesis node として持てば、Graph が「概念 → 意図 → ラベル」の流れを表現できる。

---

## 3.4 Edge-Oriented Reasoning / Context Graph

- 論文: [Exploiting Edge-Oriented Reasoning for 3D Point-Based Scene Graph Analysis](https://openaccess.thecvf.com/content/CVPR2021/html/Zhang_Exploiting_Edge-Oriented_Reasoning_for_3D_Point-Based_Scene_Graph_Analysis_CVPR_2021_paper.html)
- 論文: [Visual Reasoning Using Graph Convolutional Networks for Predicting Pedestrian Crossing Intention](https://openaccess.thecvf.com/content/ICCV2021W/AVVision/html/Chen_Visual_Reasoning_Using_Graph_Convolutional_Networks_for_Predicting_Pedestrian_Crossing_ICCVW_2021_paper.html)

### 参考点

これらの研究では、**node だけでなく edge を主要な推論対象として扱う**。  
特に EdgeGCN は、relationship を独立した表現として進化させている。

### 本研究への示唆

本研究でも、重要なのは node 単体より relation である。

例:

- `intersection_state -> left_turn_intent`
- `lane_crossing_state -> left_lane_change_intent`
- `signal_state -> trajectory_direction`
- `stop_likelihood -> stop_event`

こうした edge を「説明用」ではなく、**実際に候補 score を伝播させる主役**にする必要がある。

## 4. 本研究で具体的にどう強化するか

## 4.1 多層 graph へ拡張する

現在:

```text
concept -> label
```

提案:

```text
observation -> concept -> hypothesis -> label
```

### observation

- `trajectory_curvature`
- `lateral_offset`
- `trajectory_point_like`
- `visible_length`
- `speed_change_rate`
- `gyro_z`
- `brake`
- `blinker`
- `intersection_prior`

### concept

- `trajectory_direction`
- `speed_state`
- `lane_crossing_state`
- `intersection_state`
- `turn_intensity`
- `signal_state`

### hypothesis

- `left_turn_intent`
- `right_turn_intent`
- `left_lane_change_intent`
- `right_lane_change_intent`
- `stop_event`
- `deceleration_event`
- `acceleration_event`
- `constant_speed_event`

### label

- 11 クラス

この層を入れるだけでも、今の graph はかなり graph らしくなる。

---

## 4.2 concept を単一値ではなく soft / probabilistic にする

今の問題は、途中で

- `intersection_state = YES`
- `speed_state = STOPPED`

のように 1 値へ潰してしまうことにある。

今後は、

- `intersection_prior`
- `turn_support`
- `lane_change_support`
- `stop_support`

を 0〜1 の連続値で持ち、必要に応じて uncertainty も持たせるべきである。

### 最小実装案

`driving_graph.py` 内で、

- `concept_value`
- `concept_confidence`
- `concept_uncertainty`

を持つ辞書に変える。

例:

```python
"intersection_state": {
    "value": "UNCERTAIN",
    "confidence": 0.58,
    "uncertainty": 0.31,
}
```

---

## 4.3 edge を score propagation に使う

現状の edge は summary 用に近い。  
次は、edge を実際の推論へ使うべきである。

### 最小実装案

`supports` / `contradicts` を持つ edge に対して、

- source の score
- edge weight

を使って target hypothesis を更新する。

例:

```text
trajectory_direction=LEFT --supports(0.6)--> left_turn_intent
intersection_state=YES --supports(0.7)--> left_turn_intent
lane_crossing_state=LEFT --supports(0.7)--> left_lane_change_intent
intersection_state=YES --contradicts(0.4)--> left_lane_change_intent
```

これなら、

- 左方向変化
- 交差点あり

が集まると `left_turn_intent` が伸び、  
一方で `left_lane_change_intent` は抑制される。

---

## 4.4 topology subgraph を追加する

左右折と車線変更を分けるには、交差点と lane topology が重要である。  
そこで、概念 graph の中に **topology subgraph** を持つ。

### 候補ノード

- `lane_boundary_left`
- `lane_boundary_right`
- `adjacent_lane_left_available`
- `adjacent_lane_right_available`
- `intersection_entry`
- `intersection_type`
- `turn_restriction`

### 効果

- 左方向変化があっても、交差点 entry が弱く adjacent lane が開いていれば lane change 寄り
- 左方向変化があって、intersection entry が強ければ left turn 寄り

これは T2SG の発想にかなり近い。

---

## 4.5 Graph を final classifier ではなく reranker / verifier に限定する

重要なのは、Graph を強くしても **主経路を置き換えない** ことだと思われる。  
今後の Graph は、

- candidate reranking
- contradiction detection
- ambiguous sample verification

に限定するのがよい。

つまり、

- Ver.4 の直接分類で top-k を出す
- Graph はその top-k に対して支持・矛盾を計算する
- 曖昧ケースだけ再順位付けする

という使い方が安全である。

---

## 5. 実装優先順位

### Phase A: easiest / 効果大

1. `hypothesis node` を追加する
2. concept を soft value にする
3. top-k reranking 専用に Graph を使う

### Phase B: graph らしさを上げる

4. edge weight による propagation を入れる
5. `supports` と `contradicts` の両方向更新を入れる

### Phase C: 研究として強くする

6. topology subgraph を追加する
7. counterfactual-style edge suppression
   - `intersection=NO` なら turn hypothesis を抑える
   - `adjacent_lane_unavailable` なら lane-change hypothesis を抑える

---

## 6. 本研究にとっての最適な落としどころ

既存研究を踏まえると、本研究で次に狙うべき Graph 強化は、

- GNN をいきなり全面導入すること

ではなく、

- **多層化**
- **soft concept 化**
- **edge-based reranking**
- **topology subgraph**

である。

この順なら、

- 現在のルールベース実装を大きく壊さず
- Graph を本当に graph らしくし
- 研究上も「軽量 graph から構造化 graph への発展」と説明しやすい

---

## 7. まとめ

既存研究から見ると、Graph 強化の方向は次のように整理できる。

- **DriveLM**:
  - 多層 reasoning 構造
- **T2SG**:
  - topology-aware graph
- **CBM / ProbCBM / VCBM**:
  - soft concept / uncertainty / intervenable concept layer
- **EdgeGCN / context GCN**:
  - edge 主体の推論

したがって、本研究では Graph を

- 「特徴を並べて score を付ける補助機構」

から、

- **「concept と relation を通じて候補を再ランキングする多層 graph」**

へ進化させるのが最も自然である。
