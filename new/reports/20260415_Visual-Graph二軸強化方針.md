# Visual-Graph 二軸強化方針

**作成日**: 2026年4月15日  
**対象プロジェクト**: タクシー動画の運転行動自動アノテーション  
**目的**: 今後の改善方針を、Visual 強化と Graph 強化の二軸で整理し、研究方針として固定する

---

## 1. 結論

今後の改善方針は、**Visual 強化** と **Graph 強化** の二軸とする。

- **Visual 強化**
  - Ver.4 の主経路を強くする
  - VLM が「見れば分かる」情報を増やす
- **Graph 強化**
  - 曖昧ケースだけを補助的に検証する
  - 視覚・軌道・センサから得た判断材料の関係を整理する

一言で言えば、

**「まず見て分かるようにし、それでも迷うケースだけを graph で検証する」**

という方針である。

---

## 2. この方針にした理由

これまでの比較結果から、以下が分かっている。

1. Ver.4 のような **4フレーム + 赤軌道 + 直接分類** は強い baseline だった
2. L2M/CoT による段階的推論を全件に適用すると、
   - 視覚情報が粗い中間要約に潰れる
   - センサとの矛盾が残る
   - 最終段が文章的に再解釈して不安定になる
3. 一方で、左右折 vs 車線変更、停止 vs 減速のような **境界事例** では、判断材料の関係整理が有効である

したがって、

- 主経路は **Visual を強くする**
- Graph は **補助検証器として使う**

という役割分担が最も自然である。

---

## 3. 軸1: Visual 強化

## 3.1 目的

VLM の強みは、視覚パターンを直接読むことにある。  
そのため、まずは入力画像の中に、運転行動の判断に必要な情報を明示する。

## 3.2 強化の考え方

Visual 強化では、

- 曲がり方向
- 軌道の長さ
- 速度感
- 車線との関係
- 交差点文脈

を、できるだけ **画像上で外在化** する。

## 3.3 具体候補

- 赤軌道の先端矢印で左右方向を明示する
- 点間隔や濃淡で速度感を見せる
- 点状軌道を停止手掛かりとして安定化する
- 車線境界の推定線を重ねる
- 交差点候補領域を重ねる
- 軌道番号や時系列マーカーを追加する

## 3.4 位置付け

Visual 強化は、**主経路そのものの改善**である。  
つまり、Graph や reasoning を呼ばなくても当たる sample を増やすための軸である。

---

## 4. 軸2: Graph 強化

## 4.1 目的

Graph 強化の目的は、主経路を置き換えることではなく、

- 曖昧な候補の再順位付け
- 矛盾検出
- 境界事例の補正

を行うことである。

## 4.2 何を graph 化するか

画像そのものを graph にするのではなく、

- 視覚情報
- 赤軌道
- センサ情報

から抽出した **判断材料** を graph 化する。

例:

- `trajectory_curvature`
- `lateral_offset`
- `trajectory_point_like`
- `speed_change_rate`
- `blinker_consistency`
- `intersection_prior`

## 4.3 どう強化するか

次の方向で強化する。

1. `observation -> concept -> hypothesis -> label` の多層 graph にする
2. concept を単一値ではなく `confidence / uncertainty` 付きで持つ
3. `supports / contradicts` を説明用ではなく score 伝播に使う
4. lane / intersection の topology subgraph を入れる
5. Graph は top-k reranker / verifier に限定する

## 4.4 位置付け

Graph 強化は、**補助経路の改善**である。  
主に次の境界事例で使う。

- 左右折 vs 車線変更
- 停止 vs 減速
- 等速走行 vs 加減速

---

## 5. 二軸の役割分担

二軸の関係は次の通りである。

### Visual

- 主経路
- 全 sample に効く
- 「見れば分かる」状態を作る

### Graph

- 補助経路
- 曖昧 sample に効く
- 「見ても迷う」ケースを整理する

つまり、

**Visual は情報を強く見せる軸、Graph は関係を整理して迷いを減らす軸**

である。

---

## 6. 研究としての意味

この二軸方針により、本研究は次のように整理できる。

1. 運転行動アノテーションに必要な情報を画像上へ外在化する
2. そのうえで、視覚・軌道・センサから得た判断材料を evidence graph として整理する
3. Graph は全件の主分類器ではなく、曖昧ケースの verifier / reranker として使う

したがって、本研究の核は、

**trajectory-augmented visual prompting と multimodal evidence graph を組み合わせた behavior auto-annotation**

にある。

---

## 7. 実装順

### Phase 1

- Ver.4 baseline を再現する
- Visual 強化のみを入れて評価する

### Phase 2

- Graph を hypothesis node 付きへ拡張する
- top-k reranking 専用に使う

### Phase 3

- Visual + Graph を統合し、境界事例での改善を確認する

---

## 8. 最終方針

今後の改善は、

- **Visual を主軸**
- **Graph を補助軸**

とし、reasoning を全件へ深く適用する方向には戻らない。

したがって、今後の開発は  
**「Ver.4 を壊さずに Visual を強くし、その上で Graph を曖昧ケースの検証器として載せる」**  
という方針で進める。
