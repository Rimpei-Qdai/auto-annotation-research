# 判断材料のグラフ化は既存研究にあるか

**作成日**: 2026年4月15日  
**対象プロジェクト**: タクシー動画の運転行動自動アノテーション  
**目的**: 「視覚情報とセンサ情報から取り出した判断材料をグラフ化してアノテーションする」という発想が既存研究にどこまで存在するかを整理する

---

## 1. 結論

**完全に同じ形の既存研究は、今回確認した範囲では見当たらなかった。**

ただし、近い方向の研究は複数ある。大きく分けると以下の 5 系統である。

1. **動画 + センサから運転行動を構造化して理解する研究**
2. **構造化した特徴から maneuver / intention を推論する graphical model 研究**
3. **交通 topology や scene relation を graph 化する研究**
4. **concept / explanation を中間表現として使う研究**
5. **自動アノテーションを expert guideline や video consistency で支える研究**

つまり、

- 「判断材料をそのまま end-to-end に食わせるだけでなく、中間表現として整理する」
- 「graph / concept / explanation を使って driving reasoning を助ける」

という考え方自体には先行研究がある。

一方で、

**ego-video + 将来軌道可視化 + センサから抽出した判断材料を graph 化し、運転行動の自動アノテーションに使う**

という **組み合わせ** は、少なくとも今回確認した主要論文群では直接見つからなかった。

---

## 2. 「近いが同じではない」既存研究

## 2.1 動画 + センサから構造化した運転行動理解

- 論文: [DBUS: Human Driving Behavior Understanding System](https://openaccess.thecvf.com/content_ICCVW_2019/html/ADW/Li_DBUS_Human_Driving_Behavior_Understanding_System_ICCVW_2019_paper.html)

### 要点

DBUS は front-view video と GPS/IMU 信号から、

- 基本行動
- 意図
- 注意

を含む **structured behavior representation** を作る。

### 本研究との関係

これは「生の動画とセンサをそのまま分類器に入れるのではなく、途中で意味のある行動表現に整理する」という点でかなり近い。  
ただし、DBUS は **graph reasoning** ではなく、また **auto-annotation** を主題にもしていない。

---

## 2.2 構造化特徴から maneuver / intention を推論する graphical model

- 論文: [A Dynamic Bayesian Network for Vehicle Maneuver Prediction in Highway Driving Scenarios](https://www.mdpi.com/2079-9292/8/1/40)
- 論文: [Driver intention prediction using model-added Bayesian network](https://journals.sagepub.com/doi/10.1177/0954407020968967)
- 論文: [Driving Intention Recognition of Surrounding Vehicles Based on a Time-Sequenced Weights Hidden Markov Model for Autonomous Driving](https://www.mdpi.com/1424-8220/23/21/8761)

### 要点

これらは、

- 物理状態
- 道路構造
- 交通相互作用

といった **観測可能な特徴** を用いて、

- lane keeping
- left / right lane change

などの maneuver を確率的に推論する。

特に DBN 論文は、road structure-based features, interaction-aware features, physics-based features を分けて扱い、maneuver を中間ノードとして推論している。

### 本研究との関係

この系統は、

**「生データ -> 判断材料 -> 構造化推論 -> maneuver」**

という発想そのものにはかなり近い。  
ただし対象は主に **surrounding vehicle intention prediction** であり、

- ego-centric front video を主入力にする
- VLM を使う
- auto-annotation を行う

という点は異なる。

---

## 2.3 交通 topology や scene relation を graph 化する研究

- 論文: [DriveLM: Driving with Graph Visual Question Answering](https://arxiv.org/abs/2312.14150)
- 論文: [T2SG: Traffic Topology Scene Graph for Topology Reasoning in Autonomous Driving](https://openaccess.thecvf.com/content/CVPR2025/html/Lv_T2SG_Traffic_Topology_Scene_Graph_for_Topology_Reasoning_in_Autonomous_CVPR_2025_paper.html)

### 要点

DriveLM は、perception / prediction / planning を graph-style QA でつなぎ、人間の多段 reasoning を driving task に持ち込んでいる。  
T2SG は、lane, road signal, topology relation を明示的な scene graph として作り、下流の topology reasoning を改善している。

### 本研究との関係

この系統は、

- graph を driving reasoning に使う
- relation を明示する
- lane / intersection / turn restriction のような topology を graph にする

という点で非常に参考になる。  
ただし、これらは主に

- scene understanding
- HD mapping
- VQA / planning

が目的であり、**運転行動の自動アノテーション** が主目的ではない。

---

## 2.4 concept / explanation を中間表現として使う研究

- 論文: [Concept Bottleneck Models](https://proceedings.mlr.press/v119/koh20a.html)
- 論文: [Probabilistic Concept Bottleneck Models](https://proceedings.mlr.press/v202/kim23g.html)
- 論文: [Towards Safer and Understandable Driver Intention Prediction](https://openaccess.thecvf.com/content/ICCV2025/html/Karuppasamy_Towards_Safer_and_Understandable_Driver_Intention_Prediction_ICCV_2025_paper.html)
- 論文: [Distilling What and Why: Enhancing Driver Intention Prediction with MLLMs](https://openaccess.thecvf.com/content/WACV2026/html/Artham_Distilling_What_and_Why_Enhancing_Driver_Intention_Prediction_with_MLLMs_WACV_2026_paper.html)

### 要点

CBM 系は、入力から直接ラベルへ行くのではなく、まず **human-understandable concept** を予測し、その concept から最終ラベルを出す。  
ProbCBM はさらに concept の不確実性を扱う。  
ICCV 2025 の VCBM は、driver intention prediction に explanation concept を導入している。  
DriveXplain は visual cues から **what / why** を生成し、それを蒸留して maneuver prediction を改善している。

### 本研究との関係

これは、

**視覚とセンサから得られる判断材料を、そのまま end-to-end に潰さず、解釈可能な中間表現として残す**

という点で本研究に非常に近い。  
ただし、これらは **graph** を主役にしているわけではなく、また **annotation pipeline** でもない。

---

## 2.5 自動アノテーション研究

- 論文: [Auto-Annotation with Expert-Crafted Guidelines](https://arxiv.org/abs/2506.02914)
- 論文: [SAM2Auto: Auto Annotation Using FLASH](https://arxiv.org/abs/2506.07850)

### 要点

AutoExpert は expert-crafted guidelines を使った auto-annotation を扱う。  
SAM2Auto は video auto-annotation の consistency を重視する。

### 本研究との関係

これらは **自動アノテーション** としては非常に近い。  
ただし対象は主に

- 3D LiDAR detection
- video object annotation / segmentation

であり、**運転行動ラベルのアノテーション** ではない。  
また、判断材料を graph 化して maneuver を決める構成でもない。

---

## 3. 何が「ある」で、何が「ない」か

## 3.1 あるもの

以下は既存研究にある。

- 動画 + センサから構造化した driving behavior representation を作る
- 物理状態や道路構造を graphical model に入れて maneuver を推論する
- lane / signal / topology relation を scene graph として表す
- explanation / concept を中間表現として使う
- auto-annotation を guideline や temporal consistency で支える

## 3.2 ないもの

今回確認した範囲では、以下を **まとめて** 行う研究は見当たらなかった。

- ego-centric video
- 将来軌道の visual prompting
- 車両センサ
- そこから抽出した判断材料の graph 化
- 運転行動ラベルの auto-annotation

特に、

**「graph は classifier ではなく verifier / reranker として使う」**

という設計は、近い思想はあるが、そのまま一致する形では見つかっていない。

---

## 4. 本研究の位置付け

以上を踏まえると、本研究は次のように位置付けるのがよい。

### 4.1 完全な新規発明ではない部分

- graph reasoning
- concept bottleneck
- driver intention explanation
- auto-annotation

これらはそれぞれ先行研究がある。

### 4.2 本研究で新しく組み合わされている部分

本研究の独自性は、

**運転行動自動アノテーション** を対象に、

1. 将来軌道を visual prompt として外在化し  
2. 視覚 + センサから判断材料を抽出し  
3. それを graph / concept レベルで整理し  
4. 曖昧な maneuver 候補だけを検証・再ランキングする

という **構成の組み合わせ** にある。

言い換えると、

**「graph そのもの」が新規なのではなく、  
behavior auto-annotation のために trajectory-augmented multimodal evidence graph を使う点**

が本研究の主張しやすい新規性である。

---

## 5. 研究としての言い方

論文や指導教員向けには、次の言い方が比較的安全である。

> 既存研究には、driver intention prediction のための graphical model、traffic topology scene graph、concept bottleneck に基づく explainable maneuver prediction、ならびに object-level auto-annotation が存在する。  
> 一方で、ego-centric driving video に対して、trajectory visualization と vehicle sensor から抽出した multimodal evidence を graph 化し、behavior auto-annotation の検証・再ランキングに用いる枠組みは、少なくとも我々が確認した主要研究では直接的には扱われていない。

---

## 6. 結論

したがって、

**「視覚情報とセンサ情報から取り出した判断材料をグラフ化してアノテーションする」発想に完全に同じ既存研究は見当たらない。**

ただし、

- graphical model による maneuver inference
- scene graph による topology reasoning
- concept / explanation bottleneck

は明確な先行研究として存在するため、本研究はそれらの延長上に位置付けるのが適切である。

最も自然な主張は、

**trajectory-augmented behavior auto-annotation のための multimodal evidence graph**

という位置付けである。
