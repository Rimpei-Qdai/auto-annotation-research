**目的**: L2M/CoT 実装前の到達点を再確認し、現在の改善方針を整理する

## 1. 要約

L2M/CoT や Graph を導入する前のレポートまで遡ると、研究上の主要なベースラインは **Ver.4** である。  
Ver.4 は、

- `Qwen2-VL-2B`
- `4フレーム入力`
- `赤軌道の Visual Prompting`
- `L2M/CoT なし`

という構成で、レポート上では **51.52%** の総合精度を記録している。

一方で、その後の L2M/CoT・Graph・Graph+補助特徴系の run では、無効 run を除いても一致率は概ね **40% 前後**で推移している。  
このため、現時点では

- 「推論を複雑化したことで安定して改善した」

とは言えず、むしろ

- 「Ver.4 の強い単純ベースラインに対し、後段の reasoning が不安定さを持ち込んでいる可能性がある」

という見方が妥当である。

## 3. L2M/CoT 導入前の到達点

### 3.1 Ver.1 から Ver.4 までの流れ

既存レポートに基づくと、L2M/CoT 実装前の改善の流れは以下の通りである。

| バージョン | 主な構成 | 総合精度 |
| --- | --- | ---: |
| Ver.1 | Moondream2 + 1フレーム | 19.59% |
| Ver.2 | Qwen2-VL-2B + 4フレーム | 34.02% |
| Ver.3 | Qwen2-VL-2B + 4フレーム + 環境差 | 32.32% |
| Ver.4 | Qwen2-VL-2B + 4フレーム + 軌道描画 | 51.52% |

### 3.2 Ver.4 が重要な理由

Ver.4 は、L2M/CoT や Graph がないにもかかわらず、以下の強みを持っていた。

- 4フレームで時系列の変化をある程度見せられる
- 赤軌道により進行方向を強く視覚化できる
- 最終分類が比較的シンプルで、後段の複雑な再解釈が少ない

特にレポートでは、Ver.3 → Ver.4 の改善幅が **+19.20pt** と大きく、  
その主因は `Visual Prompting + プロンプト改善` と整理されている。

## 4. ただし Ver.4 の 51.52% をそのまま絶対視はできない

`精度改善の技術的分析_論文用.md` では、Ver.4 の改善にはサンプル分布の追い風も含まれていると分析されている。

補正後の推定では、

- Ver.4 実測: 51.52%
- 同分布補正後の推定: **40.4% ～ 45.5%**

とされている。

つまり、Ver.4 の 51.52% はそのまま現在 run と単純比較してよい値ではない。  
ただし、それでも重要なのは次の点である。

- L2M/CoT 前の単純構成でも、**少なくとも 40% 台中盤相当の力は出ていた可能性が高い**
- 今の複雑な推論系が、そのベースラインを明確に上回ったとはまだ言えない

## 5. その後の L2M/CoT・Graph 系 run との関係

既存の比較レポートから見ると、その後の run は次のような傾向だった。

- 2026-04-07: 一致率 41.67%、Macro F1 0.226
- 2026-04-08: 一致率 19.19%、Macro F1 0.094

という点である。

## 6. 現時点での解釈

### 6.1 何が起きている可能性が高いか

現状の最も自然な解釈は、次の通りである。

1. Ver.4 は「赤軌道を見せて素直に分類する」単純系として強かった
2. その後、L2M/CoT や Graph を導入することで、判断材料は増えた
3. しかし、Level 1 / Level 2 / Graph / Level 3 の間で情報の再解釈が何度も起こり、
4. 強い視覚 cue を素直に使う代わりに、矛盾した中間表現が最終判断を不安定にしている

特にこれまでの分析では、以下が繰り返し問題になっている。

- `trajectory_motion_cue` の崩壊
- `intersection_detected` の過信または全件固定
- `speed_state` と `acceleration_cause` の矛盾
- Graph 上位候補を Level 3 が上書きする
- 停止や左右折への過剰寄せ

### 6.2 なぜ L2M/CoT で精度が落ちたと考えられるか

厳密には、現時点では

- 「L2M/CoT そのものが本質的に悪い」

と証明できたわけではない。  
Ver.4 の 51.52% にはデータ分布差も含まれる可能性があり、同一評価セットでの再実験が必要である。

ただし、現在までのログと比較レポートから見ると、L2M/CoT 導入後に精度が不安定になった理由として、少なくとも次の点が強く疑われる。

#### 1. 強い視覚手掛かりを弱い中間表現に圧縮してしまった

Ver.4 では、赤軌道や複数フレームの情報をモデルが比較的直接読んで分類していた。  
一方、L2M/CoT ではこれらを

- `PARALLEL`
- `NO`
- `STRAIGHT`
- `NO_SHIFT`

のような離散的な中間記号に落としてから後段へ渡している。

このとき中間表現が崩れると、後段は元画像ではなく、その誤った要約を前提に推論してしまう。

#### 2. 段階が増えたぶん、誤りが全体へ伝播しやすくなった

現在の系は、

- Level 1
- Level 2
- Graph
- Level 3

という複数段で構成されている。

そのため、一箇所の cue 崩壊が最終ラベルまで連鎖しやすい。  
実際に、`trajectory_motion_cue` が壊れた run では左折への極端な偏りが発生した。

#### 3. 最終段が画像よりも文章要約を再解釈する構造になった

L2M/CoT 導入後の Level 3 は、

- 中間要約
- Graph summary
- センサーヒント
- ルール文

を読み直して最終判断している。

この構造では、VLM 本来の直接的な視覚認識よりも、テキスト要約の癖や矛盾の影響を受けやすい。

#### 4. センサーヒューリスティックが視覚情報を上書きしていた

比較レポートでは繰り返し、

- `speed_diff < 0` なのに `ACCELERATING`
- `brake = 1` なのに加速候補
- `intersection_detected` を強く信じて左右折へ倒す

といった矛盾が確認されている。

これは reasoning が増えたというより、

- 不安定なルール
- 不安定な中間要約

が増え、その結果として視覚 cue を壊していたと解釈できる。

#### 5. このタスクでは推論の深さより入力特徴の質の方が支配的だった

これまで精度を大きく押し上げた要因は、

- 1フレームから4フレームへの拡張
- 赤軌道の Visual Prompting

であり、どちらも入力特徴の改善だった。

一方、L2M/CoT 以降は推論段を増やしても、特徴抽出自体が不安定なため、全体精度は 40% 前後で頭打ちしている。

つまり、現時点で最も自然な解釈は、

**L2M/CoT で精度が落ちたというより、Ver.4 の強い直接視覚判断を、中間要約と後段推論が壊してしまった可能性が高い**

というものである。

### 6.3 研究上の意味

この状況は、「L2M/CoT が無意味」ということではない。  
ただし、少なくとも現時点では

- `複雑な reasoning を入れれば精度が伸びる`

という主張はできない。

むしろ、研究としては次の言い方の方が正確である。

- Ver.4 のような強い visual baseline がすでに存在する
- その上で、L2M/CoT や Graph は **境界事例の改善** を狙うべきである
- ただし、baseline の強みを壊さない設計でなければ意味がない

## 7. ここからの提案

### 7.1 第一提案: Ver.4 を固定ベースラインとして再評価する

今後の議論を安定させるため、まずやるべきことは Ver.4 の固定である。

具体的には、同一の評価セットに対して次を横並び比較する。

1. Ver.4（L2M/CoT なし）
2. Ver.4 + L2M only
3. Ver.4 + L2M + Graph
4. 現在版

これにより、

- どこで本当に精度が伸びたのか
- どこで複雑化が逆効果になったのか

を切り分けられる。

### 7.2 第二提案: 「全体精度向上」ではなく「境界クラス改善」に目的を限定する

今の reasoning 系は、全体精度を大幅に押し上げるというより、

- 停止 vs 減速
- 左折 / 右折 vs 車線変更
- 加速 / 減速 vs 等速

のような境界事例の改善を狙う方が自然である。

したがって、研究評価も

- 全体 Accuracy
- Macro F1

だけでなく、

- 右左折識別精度
- 車線変更 Recall
- 停止 / 減速の識別精度

を主指標に含めるべきである。

### 7.3 第三提案: 「Ver.4 の視覚的強さを壊さない」制約を設ける

今後の改善では、次の原則を明示的に守るべきである。

- 赤軌道由来の強い幾何 cue は、後段 reasoning が簡単に壊さない
- Graph や verifier は上書き役ではなく、矛盾検出と境界補正に限定する
- Level 3 の自由生成に任せすぎず、上位候補拘束を明示する

つまり、方向性としては

- `simple visual baseline`
  を中心に置き、
- `reasoning`
  はその周辺で補助的に使う

構成の方が安定する可能性が高い。

---

## 8. 現時点での結論

L2M/CoT 導入前まで遡ると、研究の本当の出発点は **Ver.4** である。  
Ver.4 は、4フレームと赤軌道の Visual Prompting だけで強いベースラインを作っていた。

したがって今後は、

- 「複雑な reasoning を入れること」自体を目的にしない
- まず Ver.4 を固定 baseline として再評価する
- その上で、reasoning 系は baseline を壊さず境界事例だけを改善できるかで評価する

という方針が最も妥当である。

一言でまとめると、

**今必要なのは、さらに推論を足すことよりも、Ver.4 の強い単純系を基準にして、reasoning の純増効果を正しく測ること**である。

---

## 9. 次の具体的アクション

1. Ver.4 相当の構成を git 履歴から再現する
2. 現在使っている 100 サンプル評価セットで Ver.4 を再実行する
3. 現在版と同一条件で Accuracy / Macro F1 / クラス別 Recall を比較する
4. 誤分類を
   - baseline で正解 → reasoning で悪化
   - baseline で誤り → reasoning で改善
   に分けて分析する
5. その結果を踏まえ、reasoning を残すか、縮小するか、再設計するかを決める
