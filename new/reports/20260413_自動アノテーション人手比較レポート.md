## 5. 何が改善されたか

### 5.1 左折崩壊はかなり抑えられた

2026-04-08 の run では左折が 47 件まで暴走していたが、今回は 11 件まで下がった。  
これは、`trajectory_motion_cue` をそのまま Graph に流さず、

- `raw_trajectory_motion_cue`
- `trajectory_motion_cue`
- `trajectory_motion_cue_confidence`
- `trajectory_motion_cue_consistency`

に分離し、**矛盾した cue を `AMBIGUOUS` に落とす修正が効いた**ためと考えられる。

ログでも、最新 99 件の completed sample に対して

- `raw_trajectory_motion_cue = LEFT_TURN_CUE`: 93
- `trajectory_motion_cue = AMBIGUOUS`: 93
- `trajectory_motion_cue_consistency = CONTRADICTED`: 93

となっており、**壊れた raw cue を validation で止めること自体は成功している**。

### 5.2 右折の精度は比較的よい

今回もっとも良かったクラスは `右折` で、

- Precision: 0.714
- Recall: 0.625
- F1: 0.667

だった。  
左折崩壊を止めた結果、右方向の signal / gyro / direction fallback が生きるケースでは、右折を比較的素直に拾えるようになっている。

### 5.3 等速走行と減速は実用的な水準に近づいた

- `等速走行`: F1 0.537
- `減速`: F1 0.444

であり、今回の run ではこの 2 クラスが中核になっている。  
とくに減速は recall 0.615 と高めで、`speed_diff` の信頼性補正や `DRIVER_BRAKE` の反映が効いている。

## 8. 原因分析

### 8.1 改善が効いた部分

今回の改善で最も効いたのは、**壊れた `trajectory_motion_cue` を validation で止めたこと**である。  
前回のように `LEFT_TURN_CUE` がそのまま左折へ流れなくなり、左折偏重は大きく緩和された。

### 8.2 まだ残っている本質的なボトルネック

一方で、低レベル幾何はまだ死んでいる。

- `trajectory_relation = PARALLEL`: 99 / 99
- `intersection_detected = NO`: 99 / 99
- `direction_change = STRAIGHT`: 99 / 99
- `visual_shift = NO_SHIFT`: 99 / 99

つまり現在は、

- raw cue の暴走は止めた
- しかし VLM の幾何抽出自体はほぼ全件同じ

という状態である。  
そのため、最終的な左右判定や旋回判定は、主に

- gyro
- blinker
- speed state
- Graph の fallback

に依存しており、**「映像から幾何を読んだ」精度改善にはまだ到達していない**。

### 8.3 速度変化の evidence はまだ弱い

`motion_feature_reliable=False` が 71 / 99 と多いことから、現在の `speed_diff` は大半の sample で主根拠にしづらい。  
今回の修正は「壊れた差分を使わない」方向では正しいが、次は**短い時間窓で本当に使える速度トレンドを再構築する必要**がある。

## 9. 優先度付き改善案

## 優先度 A: まず直すべきもの

### A1. Level 1 の視覚特徴を VLM 依存から外す

今回いちばん明確な結論はこれである。  
`trajectory_relation / intersection_detected / direction_change / visual_shift` が 99 件ほぼ同一値という時点で、Level 1 の幾何特徴は実質使えていない。

改善案:

- projected trajectory から `trajectory_relation` を幾何的に算出する
- 参照線との交差量から `lane_crossing_state` を算出する
- trajectory curvature と gyro から `turn_direction` を算出する
- `intersection` は VLM 補助に留め、yes/no を丸投げしない

要点:

- 研究の本筋は推論過程の改善であり、低レベル幾何は deterministic に寄せるべき
- 今回のログから見ても、**ここが最大ボトルネック**

### A2. clip 内の連続センサで速度トレンドを作る

現在は `motion_feature_reliable=False` が多数を占める。  
これは「壊れた差分を無効化する」には有効だが、そのままでは speed 系クラスが弱い。

改善案:

- clip の前後 1〜3 秒窓で `delta_v / delta_t` を再計算する
- 単発 row diff ではなく、短窓の回帰傾きで trend を出す
- `停止` と `発進` には instantaneous speed だけでなく dwell / release を入れる

### A3. 停止と発進の定義を人手ラベルに寄せる

今回もっとも大きな混同は `停止 -> 等速走行` だった。  
これは推定だが、人手の `停止` が「完全停止瞬間」より広い区間を指している可能性が高い。

改善案:

- `speed≈0` の連続フレーム長を stop evidence に加える
- brake OFF でも `speed≈0` 維持中なら stop candidate を残す
- 発進は「低速 + 正の trend + 停止直後」の組み合わせで定義する

## 優先度 B: 次に効くもの

### B1. 車線変更の専用特徴を足す

現状は `lane_crossing_state=NONE` が 94 / 99 で、左車線変更はほぼ検出不能である。

改善案:

- trajectory の横変位量
- 基準線交差の有無
- 交差量の持続時間

を明示的な feature として Graph に入れる。

### B2. contradicted cue を Level 3 でもより強く抑制する

今は Graph で contradicted cue を減点しているが、Level 3 の自然言語推論側で still left/right 解釈が復活することがある。

改善案:

- `trajectory_motion_cue_consistency=CONTRADICTED` のときは、turn / lane change を選ぶために追加 evidence を必須化する
- `PARALLEL + NO intersection + NO shift` の場合は、turn を保守的に抑える

## 10. 結論

今回の run は、**2026-04-08 時点の左折崩壊からは明確に改善した**。

- 一致率: 19.19% → 41.41%
- Macro F1: 0.094 → 0.198
- kappa: 0.057 → 0.237

改善の主因は、

- `trajectory_motion_cue` の validation
- long-lookback `speed_diff` の無効化
- Graph の contradiction handling

である。

一方で、ログからはっきり分かる通り、**Level 1 の幾何特徴抽出はまだほぼ死んでいる**。  
したがって次の本命は、prompt をさらにいじることではなく、

- trajectory / lane relation を deterministic に計算する
- clip-local な速度 trend を再構築する
- 停止 / 発進の定義を人手ラベル寄りに見直す

の 3 点である。
