## 7. 原因整理

### 原因 1. Stage2 の `B/C/D` 判定で、Qwen3-VL が `C` を全く返していない

今回の最も大きな問題はここである。

実際の raw stage2 出力は:

- `B`: 12件
- `D`: 26件
- `C`: 0件

で、右回転系に対する選択肢がモデル内で死んでいる。

### 原因 2. 右回転系 sample の一部は Stage1 ですでに `A` に吸われている

たとえば人手で `C` の sample では:

- sample `4`: Stage1=`A`
- sample `9`: Stage1=`A`
- sample `10`: Stage1=`A`
- sample `57`: Stage1=`A`

となっていた。

つまり、

- Stage2 が `C` を返さない問題
- その前に Stage1 が `A` へ吸う問題

の二重の崩れがある。

### 原因 3. `D (その他)` と `A (直線系)` の境界が不安定

`A -> D` が 18件、`D -> A` が 16件ある。

これは、

- 軌道 summary でほぼ直進に見える低速/停止系
- 元フレームでは止まって見えるが、summary では短い直進に見える系

が、Qwen3-VL の中で安定して切り分けられていないことを示す。

### 原因 4. summary 分離は有効だったが、それだけでは右左の符号を安定して読ませられていない

今回の変更で

- 元フレーム4枚
- trajectory_summary 1枚

へ分離したことで、one-shot run よりは整理された入力になった。

しかし最終的には、

- 左は多少拾える
- 右は全く出ない

ので、軌道の「符号付き方向」を VLM がまだ十分には理解していないと考えるのが自然である。

## 8. 改善案

### 改善案 A. Stage2 を `左右回転` と `その他` に分けず、まず `左右回転か / その他か` を切る

優先度: 最優先

現状の Stage2 は `B/C/D` の 3択だが、Qwen3-VL はそこから `B` と `D` しか返していない。

次は:

1. `R: 回転系` vs `D: その他`
2. `R` のときだけ `B: 左` vs `C: 右`

に分けるべきである。

これで `C` が消える原因が

- `D` との競合なのか
- 左右そのものが読めていないのか

を切り分けられる。

### 改善案 B. 右回転系だけの少数 sample でミニ評価を先に行う

優先度: 高

今回のように 100 件を一気に回す前に、

- 左折
- 右折
- 右車線変更

の代表 sample 10〜20件だけで

- Stage1
- Stage2

の raw 出力を確認した方が効率がよい。

### 改善案 C. trajectory summary に「左右方向の符号」を補助的に強く入れる

優先度: 高

現在の summary は純粋な軌道形状だけだが、Qwen3-VL が右方向を取り逃がしているため、次は

- 終点マーカーの矢印をより強くする
- 左右に薄いガイド領域を引く
- summary 上の右方向を視覚的に強く示す

など、**形状を壊さない範囲で右左の符号だけを少し強化**する余地がある。

### 改善案 D. `A/D` 境界には deterministic feature を補助で使う

優先度: 中

今回 `A <-> D` の相互誤りが非常に大きい。

ここは VLM に丸投げするより、

- speed
- visible_count
- forward distance

のような低レベル特徴を補助判定として入れた方が安定しやすい。

### 改善案 E. 4分類比較では kappa を必ず見る

優先度: 中

今回は Accuracy が少し上がっても kappa が大きく落ちた。

したがって今後の比較では、

- Accuracy だけでなく
- Macro F1
- kappa

を必ずセットで見るべきである。

## 9. 結論

今回の段階分離 run は、

- 元フレームと trajectory summary を分離した
- Stage1/Stage2 を明示的に保存した

という点で前進だった。

一方で、結果としては

- `C (右回転系)` が 0件
- Stage2 raw 出力が `B` と `D` のみ
- `A <-> D` の取り違えも大きい

という構造的な偏りが見えた。

したがって次に必要なのは、

- 3択の Stage2 をさらに二段化して原因を切り分けること
- 右回転系だけの小規模評価で raw 出力を先に観察すること
- `A/D` 境界には軽い deterministic 補助を入れること

である。
