# Prompt 修正根拠メモ

## 1. このメモの目的

最近の Qwen3-VL 系実験では、軌道描画の再設計に加えて prompt の段階分割も行っている。
一方で、prompt 修正は「なんとなく効きそう」だけで進めると、後から

- 何が文献に支えられていたのか
- 何が観測ベースの工学的調整だったのか
- どこまでが再現可能な知見なのか

が曖昧になりやすい。

そのため本メモでは、**最近の prompt 修正のうち、どこが既存研究と整合しており、どこがまだ実験仮説に留まるのか** を整理する。

---

## 2. 結論

結論から言うと、最近の prompt 分割

- `A vs 非A`
- `回転系 or その他`
- `左 or 右`

という構成は、**方向性としては既存研究と整合している**。

ただし、**この分割そのものが本タスクで最適であることを直接保証する論文はない**。

つまり、

- **分解して考えさせる**
- **一度に広い選択肢を見せない**
- **視覚入力と対応づけた短い判断をさせる**

という考え方には文献的な後ろ盾がある一方で、

- `A/N`
- `R/D`
- `B/C`

という今回の具体的な切り方は、**本研究の失敗ログから導いた工学的な設計仮説**である。

---

## 3. 直近の prompt 修正は何に基づいていたか

直近の実装では、以下の失敗パターンが観測された。

1. 一発 4 択では `A` に崩壊しやすい
2. `B/C/D` の 3 択では `C` が出ず、`B` か `D` に寄りやすい
3. 長い説明文を追加しても、VLM が軌道の符号付き幾何を安定して読めるようにはならない

この観測から、

- まず `A` とそれ以外を切る
- 次に `回転` と `その他` を切る
- 最後に `左/右` のみを判断する

という分割へ変更した。

したがって、**直近の prompt 分割の直接の根拠は実験ログの失敗パターン**であり、最初から論文からそのまま引いてきたものではない。

---

## 4. 文献的に支えられている部分

### 4.1 複雑なタスクを小さな sub-task に分解する考え方

以下の研究は、難しいタスクを一発で解かせるより、専用 prompt を持つ小問題へ分解した方が安定することを示している。

- Decomposed Prompting: A Modular Approach for Solving Complex Tasks  
  <https://arxiv.org/abs/2210.02406>

この論文が直接示すのは主に言語タスク寄りだが、

- 一度に扱う選択肢を減らす
- 中間段階を設ける
- 各段階を短い判断問題にする

という設計思想は、今回の macro 分割 prompt と整合する。

### 4.2 段階的推論・中間推論の有効性

- Chain-of-Thought Prompting Elicits Reasoning in Large Language Models  
  <https://arxiv.org/abs/2201.11903>

この研究は、複雑な結論を直接求めるより、中間推論を引き出した方が reasoning 性能が上がることを示した。

ただし本研究の運転行動タスクでは、以前の L2M/CoT で

- 中間表現の誤り固定
- 文章要約への過依存

が起きたため、今回使うべきなのは**長い自由文推論ではなく、短い分岐型の段階判断**である。

### 4.3 マルチモーダルでも compositional / structured prompt が有効

- Compositional Chain-of-Thought Prompting for Large Multimodal Models  
  <https://openaccess.thecvf.com/content/CVPR2024/html/Mitra_Compositional_Chain-of-Thought_Prompting_for_Large_Multimodal_Models_CVPR_2024_paper.html>

この研究は、マルチモーダルでも一発回答より、構造化された compositional reasoning を経由する方が有利な場合があることを示している。

今回の

- `A/N`
- `R/D`
- `B/C`

のような段階分割は、この「一度に全部決めず、構造化して判断する」方向と整合する。

### 4.4 ただし、視覚的 grounding が伴わない prompt だけでは限界がある

以下の研究は、multimodal reasoning では**長い prompt の工夫だけでなく、視覚と中間判断がきちんと grounding されること**が重要であることを示唆する。

- Argus: Vision-Centric Reasoning with Grounded Chain-of-Thought  
  <https://openaccess.thecvf.com/content/CVPR2025/html/Man_Argus_Vision-Centric_Reasoning_with_Grounded_Chain-of-Thought_CVPR_2025_paper.html>
- Interleaved-Modal Chain-of-Thought  
  <https://openaccess.thecvf.com/content/CVPR2025/html/Gao_Interleaved-Modal_Chain-of-Thought_CVPR_2025_paper.html>

この観点から見ると、本研究で prompt を細かくするだけでは不十分であり、**画像5/6の trajectory summary 自体が VLM に読みやすく設計されていること**が前提になる。

---

## 5. 文献からは直接は言えない部分

以下は、現時点では文献の直接保証ではなく、本研究の実験仮説である。

### 5.1 `A/N` という最初の分け方

文献は「分割が有効」であることを支えるが、**最初に `A/N` を切るのが最良**とまでは言っていない。

これは本研究で

- `A` への崩壊が多かった
- `A` を先に切り分けたい

という観測から導いた設計である。

### 5.2 `R/D` を先に聞いてから `B/C` を聞く順序

これも、「3 択より二段二択の方が安定するのではないか」という工学的仮説である。
文献は compositional / decomposed prompting を支持するが、**この順序が最適であることの直接根拠ではない**。

### 5.3 この分割で Qwen3-VL が軌道の左右を読めるようになるという期待

これはまだ未確証である。
実験上は summary を 2 枚に増やした後も、VLM 単体はまだ `A` 寄りであり、graph による rescue が大きい。

したがって、**prompt 分割だけで左右の符号付き幾何が読めるようになるとは言えない**。

---

## 6. 今後の prompt 修正ルール

今後 prompt を変更する際は、次の基準を守る。

### 6.1 文献的に許容される変更

- 一発多択を小さな sub-task に分ける
- 各段階の選択肢数を減らす
- 長い自由文説明より、短い二択・三択へ寄せる
- summary 画像を主に見るよう視線誘導を短く入れる

### 6.2 避けるべき変更

- 長い規則文を追加して VLM に覚えさせようとする
- 軌道から読み取るべき内容を prompt 内で言いすぎる
- 中間判断を画像側や prompt 側で決めつける
- 文献根拠なしに選択肢や段階を細かく増やし続ける

### 6.3 実装前に確認すべきこと

新しい prompt 修正を入れる前に、少なくとも以下のどれに当たるかを明示する。

1. **文献根拠あり**
   - decomposition / grounding / reduced choice load に沿っている
2. **観測ベースの仮説**
   - 現 run の失敗分布に対する工学的対策
3. **未確証**
   - 試す価値はあるが、論文的な裏づけが薄い

---

## 7. 本研究における位置づけ

本研究の現状では、

- 軌道描画の設計
- graph による幾何補正

の方が、prompt 修正より支配的に効いている。

したがって、prompt は

- 主役ではなく補助
- 画像5/6をどう見るかを短く固定する
- 一度に広い選択肢を見せすぎない

という役割に留めるのが妥当である。

一言でまとめると、

**prompt 分割の方向性自体は既存研究と整合するが、今回の具体的な段階構成はまだ本研究内の仮説であり、主改善はあくまで visual grounding 側に置くべきである。**

---

## 8. 追加メモ: stage ごとに summary だけを見せる修正について

直近の `summary_v4_prompt_balance` run では、

- `stage1_choice = A` が 96 / 96
- `stage2` と `stage3` は一度も起動しない

という結果になった。

このときの解釈として自然なのは、

- Qwen3-VL が trajectory summary を読めていない

だけではなく、

- **raw frame 4枚が先頭にあり、summary 2枚よりも文脈画像に注意が引っ張られている**

可能性である。

このため、次の prompt 修正として

- Stage1 / Stage2 / Stage3 では **summary 2枚だけ** を見せる
- raw frame は trajectory 判定から外す

という変更は、文献上も方向性として妥当である。

この修正の根拠は次の通り。

### 8.1 Vision-centric / grounded reasoning の観点

- Argus: Vision-Centric Reasoning with Grounded Chain-of-Thought  
  <https://openaccess.thecvf.com/content/CVPR2025/html/Man_Argus_Vision-Centric_Reasoning_with_Grounded_Chain-of-Thought_CVPR_2025_paper.html>

- Interleaved-Modal Chain-of-Thought  
  <https://openaccess.thecvf.com/content/CVPR2025/html/Gao_Interleaved-Modal_Chain-of-Thought_CVPR_2025_paper.html>

これらは、multimodal reasoning では「長く説明すること」よりも、
**どの visual input を主に読ませるかを明確にすること**が重要だと示唆する。

今回の stage 分割では、Stage1-3 の問いはすべて trajectory 幾何そのものを問うている。
したがって、trajectory summary だけに入力を絞るのは、視覚的 grounding を強める方向として自然である。

### 8.2 選択肢だけでなく入力も絞る

- Decomposed Prompting: A Modular Approach for Solving Complex Tasks  
  <https://arxiv.org/abs/2210.02406>

この観点を本研究に引きつけると、分解すべきなのは「選択肢」だけではない。
各 sub-task に不要な入力も減らし、

- Stage1 は summary の幾何だけ
- Stage2 は summary の回転/停止らしさだけ
- Stage3 は summary の左右だけ

に寄せる方が、task decomposition として一貫している。

### 8.3 ただし、この修正もまだ本研究内の仮説である

文献は「関係ない画像を外せば必ず今回の Qwen3-VL が改善する」とまでは保証しない。
したがって、この修正は

- **文献的な方向性には沿う**
- **しかし本タスクでの有効性はまだ検証が必要**

という位置づけである。

## 9. 追加メモ: direction summary と speed summary を分ける修正について

`summary_v6_direction_speed_split` では、

- 画像1: 左右の曲がりだけを読む `direction summary`
- 画像2: 前進距離と点間隔だけを読む `speed summary`

へ役割を分離した。

この変更の狙いは、1枚の summary に

- 軽い曲率
- 明確な右左折
- stop-like な短さ
- 等速 / 減速 / 加速の差

を同時に載せないことである。

この修正の根拠は次の通り。

### 9.1 MLLM は小さい差分や複数 cue の混在に弱い

- Exploring Perceptual Limitations of Multimodal LLMs on Small Visual Objects  
  <https://openreview.net/forum?id=D8MjYW8m35>

- MLLMs Know Where to Look  
  <https://openreview.net/forum?id=DgaY5mDdmT>

これらは、MLLM が

- 小さな視覚差分
- distractor の多い入力
- 同時に複数の視覚課題を解かせる状況

で不安定になりやすいことを示している。

### 9.2 visual prompt には task semantics の分離が必要

- Exploring the Transferability of Visual Prompting for Multimodal Large Language Models  
  <https://openaccess.thecvf.com/content/CVPR2024/html/Zhang_Exploring_the_Transferability_of_Visual_Prompting_for_Multimodal_Large_Language_CVPR_2024_paper.html>

この研究の示唆は、視覚 prompt は単に目立てばよいのではなく、
**「何を見る画像なのか」** が task semantics として明確である方がよい、という点にある。

本研究に引きつけると、

- `direction summary`: 左右の符号と曲率を見る
- `speed summary`: 前進距離と点間隔を見る

と役割を分ける方が自然である。

### 9.3 この修正の位置づけ

したがって、この修正は prompt を複雑にするためではなく、
**VLM に一度に1種類の視覚課題だけを与えるための visual grounding 強化**
として位置付けられる。
