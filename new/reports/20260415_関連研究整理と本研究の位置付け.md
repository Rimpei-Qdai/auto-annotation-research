# 関連研究整理と本研究の位置付け

**作成日**: 2026年4月15日  
**対象プロジェクト**: タクシー動画の運転行動自動アノテーション  
**目的**: 自動アノテーション関連研究を整理し、本研究がどこに位置付くかを明確化する

---

## 1. 背景

本研究では、タクシーの車載動画とセンサデータを用いて、

- 等速走行
- 加速
- 減速
- 停止
- 発進
- 左折 / 右折
- 車線変更

などの**運転行動ラベルを自動付与する**ことを目的としている。

ここで重要なのは、本研究の対象が

- 物体検出
- セグメンテーション
- 3D cuboid annotation

ではなく、**短いクリップに対する行動ラベル付与**である点である。  
この点で、自動運転向け auto-annotation の既存研究と近い部分はあるが、完全には一致しない。

---

## 2. 関連研究

## 2.1 Auto-Annotation with Expert-Crafted Guidelines

- 論文: [Auto-Annotation with Expert-Crafted Guidelines: A Study through 3D LiDAR Detection Benchmark](https://arxiv.org/abs/2506.02914)
- 出典: arXiv 2506.02914

### 要点

この研究は、**人手 annotator が従う expert-crafted guideline を活用して auto-annotation を行う**という発想を取っている。  
対象は nuScenes を再構成した `AutoExpert` ベンチマークであり、LiDAR 上の 3D cuboid annotation を扱う。

パイプラインは概念的に単純で、

1. RGB 画像上で 2D detection / segmentation
2. それを既知のセンサ姿勢で 3D に持ち上げる
3. 3D cuboid を生成する

という流れで expert guideline に沿った annotation を自動化している。

### 本研究への示唆

この研究から最も参考になるのは、**「annotation guideline 自体をモデルやパイプラインに組み込む」**という考え方である。  
本研究でも、単に VLM に行動ラベルを当てさせるのではなく、

- 停止と減速の違い
- 左右折と車線変更の違い
- どの証拠を重視するか

を guideline として明示し、それに沿って自動アノテーションする設計が有効である。

### 限界

一方で、この研究は **3D object annotation** が対象であり、運転行動のような clip-level action labeling は扱っていない。  
したがって本研究では、

- guideline-driven auto-annotation の考え方

は借りられるが、

- 行動分類そのものの解決策

としては再設計が必要である。

## 2.2 SAM2Auto: Auto Annotation Using FLASH

- 論文: [SAM2Auto: Auto Annotation Using FLASH](https://arxiv.org/abs/2506.07850)
- 出典: arXiv 2506.07850

### 要点

この研究は、**人手介入や dataset-specific training を必要としない fully automated video annotation pipeline** を提案している。  
中核は、

- `SMART-OD`
  - 自動 mask 生成 + open-world object detection
- `FLASH`
  - フレーム間で一貫した object identity を維持する video instance segmentation

であり、動画全体を通した annotation consistency を重視している。

### 本研究への示唆

この研究で重要なのは、**単フレーム単位ではなく、動画全体で一貫した annotation を保つこと**である。  
本研究でも、

- 1フレームごとの見え方
- その瞬間の VLM の気分

でラベルが揺れるのではなく、

- 5秒クリップ全体
- 4フレーム間の一貫性
- 軌道の連続性

を重視した設計が必要である。

### 限界

ただし、SAM2Auto の主対象は **物体検出・追跡・セグメンテーション** であり、  
本研究のような

- 停止 vs 減速
- 左折 vs 車線変更

といった**行動ラベル境界**の問題とは異なる。

したがって本研究では、

- temporal consistency を保つ考え方

は参考になるが、

- そのまま使えるモデルではない

と整理するのが適切である。

## 2.3 Semi-Automatic Cloud-Native Video Annotation for Autonomous Driving

- 論文: [Semi-Automatic Cloud-Native Video Annotation for Autonomous Driving](https://www.mdpi.com/2076-3417/10/12/4301)
- 出典: Applied Sciences 2020, 10(12), 4301

### 要点

この研究は、**自動運転向け動画 annotation をクラウドネイティブに大規模運用する仕組み**を提案している。  
中核は `Annotation as a Service (AaaS)` であり、

- 自動 pre-annotation
- 人手 review / correction
- Kubernetes ベースの大規模処理

を統合している。

論文では、automatic pre-annotation によって人手作業時間を大きく削減できることが示されている。

### 本研究への示唆

この研究は、**完全自動で100%正解を狙うより、まず高品質な pre-annotation を作る**という立て方が実用上有効であることを示している。  
本研究でも、

- 完全自動の最終ラベル生成

だけを目指すのではなく、

- 高品質な自動下書き
- 人手修正コストの削減

という位置付けを取ることは十分に妥当である。

### 限界

この研究は主に**annotation infrastructure / workflow** の貢献であり、  
本研究が扱うような

- VLM による行動ラベル判断
- 軌道描画
- 4フレーム時系列入力

のような、判断そのものの精度改善には踏み込んでいない。

## 2.4 Efficient 3D object annotation via pseudo-LiDAR and VLM validation

- 論文: [Efficient 3D object annotation via vision-derived pseudo-LiDAR and Vision Language Model (VLM) validation](https://www.sciencedirect.com/science/article/pii/S0968090X25004334)
- 出典: Transportation Research Part C, 2026

### 要点

この研究は、

- pseudo-LiDAR を生成し
- pseudo-label を作り
- VLM で validation / filtering する

という**closed-loop auto-annotation** を提案している。

### 本研究への示唆

ここで特に有用なのは、**VLM を主生成器ではなく validator として使う**発想である。  
本研究でも、VLM にすべてを決めさせるより、

- まず baseline が候補を出す
- 曖昧ケースだけ verifier が確認する

方が安定しやすい可能性が高い。

### 限界

この研究も対象は 3D object annotation であり、行動分類ではない。  
したがって本研究では、

- VLM as verifier

という役割設計だけを借りるのが適切である。

---

## 3. 関連研究全体から見える整理

上記の研究をまとめると、自動運転向け auto-annotation 研究は大きく次の4方向に分けられる。

### 3.1 guideline-driven auto-annotation

- 代表: Auto-Annotation with Expert-Crafted Guidelines
- 主眼:
  - annotation guideline を使って自動化する

### 3.2 temporal consistency / video-wide consistency

- 代表: SAM2Auto
- 主眼:
  - フレーム単位ではなく動画全体で一貫した annotation を作る

### 3.3 semi-automatic human-in-the-loop

- 代表: Semi-Automatic Cloud-Native Video Annotation for Autonomous Driving
- 主眼:
  - pre-annotation を自動化し、人手修正コストを下げる

### 3.4 VLM-based validation

- 代表: Efficient 3D object annotation via pseudo-LiDAR and VLM validation
- 主眼:
  - VLM を annotation validator / filter として使う

---

## 4. 本研究の位置付け

本研究は、上記研究群と比べると、次の点で独自性がある。

### 4.1 対象タスクが「物体 annotation」ではなく「運転行動 annotation」

既存研究の多くは、

- 2D/3D object detection
- segmentation
- cuboid annotation

を対象としている。

一方、本研究の対象は、

- 4フレーム + センサ + 赤軌道

から、

- 停止 / 減速 / 等速 / 加速
- 左折 / 右折
- 車線変更

といった**運転行動ラベル**を付与することである。

つまり本研究は、**scene/object annotation ではなく maneuver / behavior annotation** に位置付く。

### 4.2 赤軌道 Visual Prompting を中核にしている

本研究では、速度とヨーレートから 3 秒先軌道を描画し、

- 赤軌道の曲がり
- 赤軌道の長さ
- 赤軌道の横偏位

を視覚 evidence として使っている。

この点は、既存の object-centric auto-annotation 研究とは異なり、**行動推定のための視覚的中間表現を直接設計している**と言える。

### 4.3 行動ラベルの境界事例を中心課題にしている

本研究の難所は、

- 停止 vs 減速
- 左右折 vs 車線変更
- 等速 vs 加減速

のような**意味的に近い行動の分離**にある。

この点は、物体の有無や位置を決める annotation とは異なり、  
**中間的・曖昧な行動概念の扱い**が本質になる。

---

## 5. 本研究はどう立てるべきか

上記の関連研究を踏まえると、本研究は次のように位置付けるのが最も自然である。

### 5.1 研究の基本立場

本研究は、

- **guideline-driven**
- **trajectory-augmented**
- **behavior annotation**

の研究として位置付ける。

つまり、

- expert-like annotation guideline を明示し
- 赤軌道を含む visual evidence を使い
- clip-level driving behavior を自動付与する

研究である。

### 5.2 既存研究との差分

既存研究との差分は、次のように整理できる。

- 既存:
  - 物体 annotation が中心
  - VLM は object validation や open-world detection に使われる
- 本研究:
  - 行動 annotation が中心
  - VLM は軌道と時系列を統合した行動判断に使われる

### 5.3 今後の設計方針との接続

これまでの失敗分析を踏まえると、本研究は

- 全件 L2M/CoT
- 全件 Graph

の方向よりも、

- **Ver.4 baseline-first**
- **deterministic evidence**
- **曖昧ケースだけ verifier**

の方向へ進むのが適切である。

このとき関連研究との接続は次のようになる。

- **AutoExpert 系**:
  - annotation guideline の設計に活かす
- **SAM2Auto 系**:
  - clip-wide consistency の考え方に活かす
- **AaaS 系**:
  - 実運用上の pre-annotation + human review に活かす
- **VLM validation 系**:
  - VLM を verifier として限定的に使う設計に活かす

---

## 6. 提案する本研究の最終的な位置付け

本研究は、次のようにまとめられる。

**本研究は、車載動画とセンサデータに基づく運転行動自動アノテーションを対象とし、赤軌道 Visual Prompting と expert-like guideline を用いて clip-level maneuver annotation を行う研究である。既存の object-centric auto-annotation と異なり、行動ラベルの境界事例を中心課題とし、Ver.4 の強い visual baseline を主経路に据えたうえで、deterministic evidence と verifier を補助的に用いる点に特徴がある。**

---

## 7. 結論

今回調べた論文群は、いずれも本研究に直接そのまま使えるわけではないが、非常に参考になる。

特に借りるべきなのは、

- guideline を明示すること
- 動画全体で整合性を取ること
- pre-annotation と人手修正の関係を整理すること
- VLM を verifier として使うこと

である。

一方で、本研究独自の中心課題は、

- 運転行動の意味的境界
- 軌道とセンサの統合
- 行動ラベル付与の説明可能性

にある。

したがって、本研究は既存 auto-annotation 研究の単純な応用ではなく、  
**自動運転における行動 annotation のための VLM ベース auto-annotation 研究**として位置付けるのが妥当である。
