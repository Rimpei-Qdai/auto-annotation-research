# グラフ生成

## 1. 入力

Graph 生成には次の3種類の情報を使う。

- **Level 1 の出力**
  - `trajectory_motion_cue`
  - `trajectory_relation`
  - `intersection_detected`
  - `direction_change`
  - `visual_shift`
  - `road_shape`
- **Level 2 の出力**
  - `speed_trend`
  - `acceleration_cause`
  - `consistency_check`
- **センサ + 赤軌道から得た deterministic feature**
  - `speed`, `speed_diff`, `speed_change_rate`, `timestamp_diff_sec`
  - `brake`, `gyro_z`, `blinker`
  - `trajectory_point_like`, `trajectory_visual_speed_state`
  - `trajectory_lateral_offset_m`, `trajectory_heading_delta_rad`, `trajectory_curvature_score`

## 2. 生成手順

Graph 生成は概ね次の順で行う。

1. **概念ノード生成**
   - `speed_state`
   - `signal_state`
   - `trajectory_direction`
   - `turn_intensity`
   - `lane_crossing_state`
   - `intersection_state`
   - `stop_likelihood`
   などを生成する

2. **エッジ生成**
   - `supports`
   - `contradicts`
   - `supports_turn`
   - `supports_lane_change`
   - `weak_turn_context`
   を付与する

3. **ラベル支持度の計算**
   - 各概念から 11 クラスに加点・減点する
   - 例:
     - `intersection=YES` なら左右折を加点
     - `intersection=NO` かつ `lane_crossing=LEFT/RIGHT` なら車線変更を加点
     - `stop_likelihood=HIGH` なら停止を強く加点

4. **候補抽出**
   - 上位 3 候補を `top_candidates` として保持
   - 1位の score が十分高く、2位との差が大きいときだけ `strong_candidate` とみなす

## 3. 制約条件

Graph 生成では、主に以下のような閾値・制約を置いている。

- `speed_diff` や `speed_change_rate` による加減速判定
- `timestamp_diff_sec` が大きい場合、運動特徴の信頼性を下げる
- `trajectory_point_like=True` かつ低速のとき、停止寄り evidence とみなす
- `gyro_z` や `trajectory_curvature_score` が大きいと、turn evidence を強める
- `signal_state` と `trajectory_direction` が一致すれば support、不一致なら contradict
- `intersection_state=YES` は turn を支持し、`intersection_state=NO` は lane change を支持する

## 4. どこで使われるか

生成された Graph は、最終的に以下の2箇所で使われる。

- **Level 3 への入力**
  - Graph 概念要約
  - 上位候補
  - strong candidate
  をプロンプトに渡す
- **後処理**
  - 右左折と車線変更の競合
  - 停止と減速の競合
  - speed-event と turn-event の競合
  を補正する

## 5. 現状の限界

現状の Graph は、名前としては Graph だが、実態は **「関係付き特徴列挙 + スコアリング」** に近い。  
つまり、

- ノード間で本格的な情報伝播をしているわけではない
- 学習された graph inference ではない
- 最終判定を構造的に強く拘束する仕組みもまだ弱い

したがって、現状の Graph は **真の graph reasoning というより、候補整理・矛盾検出・再ランキングのための軽量補助機構** と位置付けるのが適切である。
