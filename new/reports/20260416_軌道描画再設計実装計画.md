# 軌道描画再設計 実装計画

## 1. 結論

今後の改善は、**プロンプトの言い換えを主軸にするのではなく、VLM が誤読しにくい軌道描画へ再設計することを主軸**に進める。  
プロンプトはその描画の意味を短く固定する補助とし、長いルールの追加は優先しない。

理由は次の通り。

- これまで prompt を調整しても `1`, `A`, `L` などへの崩壊は根本的には解消しなかった
- 一方で、元フレームと `trajectory summary` の分離や graph 補正は一定の改善を生んだ
- 最新の Qwen3-VL + graph 実験でも、**VLM 自体は右回転系 `C` を直接出せていない**
- したがって、現状のボトルネックは「どう聞くか」より「何を見せているか」にある

## 2. 今回の方針

今後は以下の役割分担で進める。

- **軌道描画**: 左右方向、前進距離、曲率、停止っぽさを画像だけで自明にする
- **プロンプト**: どの画像を主に見るか、どの順番で見るかだけを短く固定する
- **graph / deterministic 幾何**: VLM が苦手な符号付き幾何の補助検証に使う

つまり、

- VLM に「理解させるために長い説明を書く」のではなく
- VLM が「見れば分かる」入力へ寄せる

のが主眼である。

## 3. 実装の柱

### 3.1 軌道描画の再設計

現在の課題は、trajectory summary がまだ「線がある画像」に近く、左右や停止っぽさが安定して伝わっていないことにある。  
次の描画方針へ切り替える。

#### A. summary は「軌道の意味」だけに絞る

- 背景なし
- 小さい文字なし
- 不要な legend なし
- sensor の解釈済みラベルなし

残す情報は最小限にする。

- 直進基準
- 軌道本体
- 始点
- 終点
- 時間順

#### B. 方向と速度感を別の visual cue に分離する

今までは 1 本の線に意味を詰め込みすぎていた。次は以下を分離して示す。

- **方向**
  - 終点方向
  - 左右偏位
  - 曲率
- **速度感**
  - 前進距離
  - 点間隔
  - 軌道全長
  - 停止時は極短であること

#### C. summary を 2 パネル構成にする

次の 2 種類を同時に見せる。

1. **Top-down trajectory summary**
   - 直進基準と予測軌道を上から見せる
2. **Normalized geometry summary**
   - 原点中心・固定スケールで左右差だけを強調する

目的は、元フレームに依存しない符号付き幾何を VLM に安定して見せること。

#### D. 右左の符号を絵として固定する

現在は右回転が弱い。  
そのため、右左の違いが視覚的に完全に対称になるようにする。

- 画像座標と summary 座標の右左を明示的に一致させる
- 右回転なら summary でも必ず右側へ伸びる
- 左回転なら summary でも必ず左側へ伸びる

ここは実装後にサンプルで必ず目視確認する。

### 3.2 プロンプトの簡素化

プロンプトは次の役割だけに限定する。

- 画像1〜4は元フレーム
- 画像5〜6は軌道要約
- 主に要約画像を見る
- まず直線系かどうか、その後に左右/その他を判断する

入れないもの:

- 長いルール列挙
- センサーからこちらが解釈した中間ラベル
- 「〜なら必ず〜」のような強すぎる文

### 3.3 評価の切り分け

今後は毎回、次の 3 条件を分けて比較する。

1. **VLM only**
   - 元フレーム + 新 summary
2. **VLM + graph**
   - 上記 + graph rerank
3. **deterministic only**
   - 幾何 + センサーのみ

これで、

- 描画だけでどこまで改善したか
- graph がどれだけ上積みしたか
- VLM を使う価値がどこにあるか

を切り分ける。

## 4. 実装フェーズ

### Phase 0. ベースライン固定

目的:
- 比較の土台を固定する

作業:
- 現在の `codex/qwen3-vl-graph` を基準 run とする
- 最新 metrics を比較表に固定する
- representative sample を 12 件ほど固定する

代表 sample のカテゴリ:
- 直線系 正例
- 左回転系 正例
- 右回転系 正例
- その他 正例
- 現在の誤分類代表

### Phase 1. 軌道描画の再設計

目的:
- VLM が summary だけで左右と停止っぽさを読みやすくする

作業:
- `visual_prompting.py` を再設計
- `trajectory summary` を 2 パネル化
- fixed scale / fixed orientation を導入
- 小さい文字や解釈済みラベルを削除

完了条件:
- 代表 sample で右左が人間目視で一貫して読める
- 停止 sample が「極短軌道」として見える
- 直線 sample が不要に曲がって見えない

### Phase 2. プロンプト簡素化

目的:
- VLM に「どこを見るか」だけを伝える

作業:
- stage prompt を短く書き直す
- sensor の生値は残すが、解釈済み cue は削る
- 出力形式は今のまま維持

完了条件:
- prompt token 数が減る
- raw output が安定する

### Phase 3. VLM only 再評価

目的:
- 描画再設計だけの効果を測る

作業:
- graph を切った run を実行
- 4分類 accuracy / macro F1 / kappa を比較
- `C` が VLM 単体で出るか確認

成功条件:
- `C` を VLM が 0 件ではなく返す
- `A -> D` が現状より減る

### Phase 4. graph 再統合

目的:
- VLM が苦手な境界だけを graph で補う

作業:
- graph を再有効化
- `A/D` 境界
- `B/C` rescue
- mild drift の過剰補正抑制

完了条件:
- VLM only より 4分類が改善
- 誤り分布が「救済」に偏り、単なる崩しが減る

### Phase 5. 11分類 fine split

目的:
- macro 4分類の上に細分類を載せる

作業:
- `A -> 1/2/3`
- `B -> 6/8`
- `C -> 7/9/10`
- `D -> 0/4/5`

を deterministic に分ける

完了条件:
- macro を崩さずに 11分類 Accuracy が上がる

## 5. 具体的な修正対象

### `new/annotation_tool/visual_prompting.py`

実施内容:
- 新しい summary renderer
- 2 パネル構成
- fixed scale / fixed orientation
- 右左が対称になる描画

実施内容:

- prompt の差し替え
- 保存ログに summary 種別を残す

### `new/annotation_tool/config.py`

実施内容:
- stage prompt の簡素化
- 表示仕様の説明更新

### `new/annotation_tool/tests/`

実施内容:
- summary 生成の左右一貫性テスト
- stop-like summary の見え方に関する最小テスト
- graph rescue 条件の回帰テスト

## 6. 今回やらないこと

次のものは、いったん後回しにする。

- VLM をさらに別モデルへ増やすこと
- scene graph の大規模再設計
- L2M/CoT 系の再導入
- sensor 種類の再拡張

まずは **入力の見せ方の再設計** を終えてから比較する。

## 7. 評価指標

毎回必ず出す。

- 11分類 Accuracy / Macro F1 / kappa
- 4分類 Accuracy / Macro F1 / kappa
- 分布
- 主な confusion
- VLM raw 出力傾向
- graph override 件数

特に重視する確認項目:

- `C` を VLM 単体で返せるか
- `A -> D` が減るか
- `A -> B/C` の過剰補正が減るか

## 8. 期待される成果

この計画の狙いは、細かい閾値調整ではなく、**VLM が軌道を誤読しにくい入力へ作り直すこと**にある。  
ここが改善しない限り、prompt も graph も対症療法になりやすい。

したがって今回の成功条件は、

- 「精度が少し上がること」だけではなく
- **VLM 単体の出力分布が崩壊しなくなること**
- **左右の符号付き幾何を raw output で使えていること**

とする。
